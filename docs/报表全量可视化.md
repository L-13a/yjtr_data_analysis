# basic.sql 全部报表可视化方案

## 报表清单与可视化方式

basic.sql 中共 13 个主报表（部分含子报表），按以下方式可视化：

- **T1 销售客单按时段** — 柱状图+折线图（X轴=时段，Y轴=销售额/客流/客单），调用存储过程 `up_rpt_io_sale_bytime`
- **T2 总客流日报** — 表格（已完成，保留现有）
- **T3 品类销售分析** — 表格（含销售额、毛利率、动销率、占比等），使用 `cross apply dbo.io_get_gds_gcode_cnt` + `tbs_d_supp`
  - T3.1 不动销商品明细 — 表格（点击 T3 行展开/跳转）
  - T3.2 动销商品明细 — 表格（点击 T3 行展开/跳转）
- **T4 销售时段对比** — 双系列柱状图（本期 vs 对比期），查 `tb_o_sg` 按小时分组两段日期对比
- **T5 畅销商品缺货** — 表格（库存 < 日均销售 x 预警天数），查 `tb_gdsstore` + `tb_gds` + `tb_gdsclass`
- **T6 商品销售分析** — 表格，查 `tbs_d_gds` + `tb_gdsstore` + `tb_gds`
- **T7 供应商到货率汇总** — 表格（含到货量率/到货额率），查 `tbs_d_gds` + `tb_o_i` + `tb_o_ig` + `tb_gdsprovider`
- **T8 负毛利** — 表格（筛选毛利 < 0 的商品），查 `tbs_d_gds` + 退货临时表
- **T9 高库存(低周转)** — 表格，查 `tb_gdsstore` + 近一年 `tbs_d_gds` 计算周转天数
- **T10 新品报表** — 表格，查 `tb_gdsstore` 中试销期商品
- **T11.1 供应商动销率** — 表格，查 `tb_gdsprovider` + `tbs_d_gds`
- **T11.2 品类动销率** — 表格，使用 `cross apply dbo.io_get_gds_gcode_cnt`
- **T12 品态异常** — 表格，查 `tb_gdsstore` 中 `c_store_status <> '正常商品'`
- **T13 滞销商品** — 表格（含销量/销额排名百分位），查 `tbs_d_gds` 用 `DENSE_RANK` 分组排名

## 前端结构（report2.html）

一个页面，顶部标签页栏切换各报表：

```
[T1 销售客单] [T2 总客流] [T3 品类分析] [T4 时段对比] [T5 缺货] [T6 商品销售] ...
```

- 点击标签页才加载对应 API（懒加载，避免一次请求全部）
- T1、T4 用 ECharts 图表 + 下方数据表格
- 其余用纯表格，金额列自动格式化（超1万显示x.xx万）
- 毛利率/占比列自动格式化为百分比

## 后端结构（app.py）

新增 12 个 API（T2 已有），每个对应一个报表：

| 路由 | 报表 | SQL 特点 |
|------|------|----------|
| `/api/t1_sale_bytime` | T1 | 调用 `exec up_rpt_io_sale_bytime` |
| `/api/t2_daily_flow` | T2 | 已有（重命名 `/api/daily_flow`） |
| `/api/t3_category_analysis` | T3 | `cross apply` + CTE |
| `/api/t3_1_no_sale_detail` | T3.1 | 子表 |
| `/api/t3_2_sale_detail` | T3.2 | 子表 |
| `/api/t4_hour_compare` | T4 | 两段时间对比 |
| `/api/t5_shortage` | T5 | 库存预警 |
| `/api/t6_product_sale` | T6 | 商品销售汇总 |
| `/api/t7_supplier_delivery` | T7 | 供应商到货率 |
| `/api/t8_negative_profit` | T8 | 负毛利筛选 |
| `/api/t9_high_inventory` | T9 | 周转天数计算 |
| `/api/t10_new_product` | T10 | 新品试销期 |
| `/api/t11_supplier_sale_rate` | T11.1 | 供应商动销率 |
| `/api/t11_2_category_sale_rate` | T11.2 | 品类动销率 |
| `/api/t12_status_abnormal` | T12 | 品态异常 |
| `/api/t13_slow_moving` | T13 | 滞销排名 |

每个 API 内部：
- 用 `get_conn()` 获取连接（和 app_1.py 一致）
- SQL 直接翻译自 basic.sql，参数先用固定默认值（全部机构、最近7天）
- 用 `cached_query()` 做 5 分钟缓存
- 返回 JSON

## 需要特别处理的 SQL

1. **T1** — 调用存储过程：`cursor.execute("EXEC up_rpt_io_sale_bytime ?, ?, ?, ?, ?, ?, ?, ?, ?", ...)`
2. **T3** — `cross apply dbo.io_get_gds_gcode_cnt(...)` 直接在 SQL 字符串中使用（数据库已有该函数）
3. **T8** — 原 SQL 用了临时表 `#temp01`，需要在**同一个连接**内先 `SELECT INTO #temp01`，再执行主查询
4. **T13** — 用 `DENSE_RANK()` 窗口函数 + CTE，直接写在 SQL 字符串中
